# 亿驾 v3.0+ 客户端交互接口文档


# 一、 背景说明

  本接口文档服务于新版亿驾客户端(`v3.0`及以上版本)，与 `v3.0` 文档的主要区别在于去除服务端分配 `user-id` 的必要条件，登录同步非登录状态下产生的用户数据。

# 二、接口设计原则

## 1. 只支持`HTTP`协议的`POST`请求

## 2. 接口地址组成

```
http://api.carbit.com.cn/{version}/{api_name}.{format}
```
* `version`, 接口版本号, **检查更新** 接口不传递此参数;
* `api_name`, 接口名称;
* `format`, 接口返回内容格式，当前支持`json`, `xml`。

`URL`示例
```
http://api.carbit.com.cn/v3.0/record.json
```

## 3. 请求内容格式

  请求内容均在`Request Body`中记录，以`json`格式。

## 4. 返回内容格式

  返回内容包含`code`，`message`和`context`，格式依`format`参数而定

  返回示例(`json`)
  ```
  {
      "code": 0,          # 返回状态码请参看状态码说明列表
      "message": "OK",
      "context": NULL     # 扩展内容
  }
  ```

## 5. 签名加密方式
* 签名方式

  在`HTTP`请求`header`头中传递`X-SIGN`参数置，用于服务端校验。

  `X-SIGN`生成方式：

  时间戳精确到：秒，未登录状态下`TOKEN`为空，也可不传递`X-TOKEN`
  ```  
  X-SIGN = strtoupper( md5( TOKEN + IMEI + app_key + URI + timestamp ) ),timestamp
  ```
  `URI`示例
  ```
  /v3.0/record.json
  ```
* 内容加密方式

    `Request Body`中的内容均需要经过加密

## 6. 终端类型加密方式
* 终端类型加密

    在`HTTP`请求`header`头中传递`X-BIZ`参数配置，用于服务端校验。

    `X-BIZ`对应参数：android、ios (后续可添加wince等)
    ```  
    X-BIZ = android
    ```

## 7. `Token`

   客户端请求登录成功后，服务器段分配客户端一个`X-TOKEN`。
   在`HTTP`请求`header`头中传递`X-TOKEN`参数配置，用于验证客户端操作的合法性。
   客户端退出后`X-TOKEN`为空。

## 8. 接口单位统一说明

* **速度单位** - 米/秒，`m/s`
* **时间** - 秒级时间戳，`January 1 1970 00:00:00 GMT`，(`Java`毫秒时间戳需要额外处理)
* **时间长度** - 秒, `s`
* **距离单位** - 米，`m`
* **方向单位** - 距离正北方角度(`0 - 359`)


# 三、参数说明及请求示例

## 1. 必传参数
* 在`HTTP`请求`header`头中传递`X-DEVICE`参数配置，用于传递设备基本信息。
* `device` 设备信息，内容均需要经过加密。

  ```
    {
        "uuid" : "ffffffff-e8bc-7dfa-dfb6-31b10033c587", # 用户标识
        "model": "Samsung I9505 Galaxy S4",   # 设备型号
        "resolution": "1080x1920",            # 屏幕分辨率
        "ppi": 441,                           # 像素密度
        "os": "4672",                         # 系统版本，数字类型
        "mac": "1F3D2EF4",                    # 无线网上物理地址
        "imei": "490154203237512",            # 在文档中又名 udid，android 中为 imei，iOS 中为 udid。设备唯一标识
        "imsi": "404685505601234",            # 国际移动用户识别码
        "phone_num": "13888888888",           # 手机号码
        "carrier": "cmcc",                    # 运营商
        "bluetooth": "4.0",                   # 蓝牙版本号
        "area": "wuhan",                      # 所在地区
        "network": "wifi",                    # 网络类型
        "location": {
            "longitude": 114.51005,           # 位置信息：经度
            "latitude": 30.561933             # 位置信息：纬度
        }
    }

  ```
* 在`HTTP`请求`header`头中传递`X-CLIENT`参数配置，用于传递设备基本信息。
* `client` 客户端信息，内容均需要经过加密。


  ```
    {
        "package_name": "com.carbi.easydrive", # 客户端包名称
        "version_name": "v3.0.1",              # 客户端版本名称
        "version_code": "32",                  # 客户端版本编号
        "channel": "B3"                        # 渠道标识
    }
  ```

## 2. 扩展参数

扩展参数定义在`context`内，不同的功能接口的子属性不同，如日志记录接口`record`的扩展参数为`actions`

# 六、客户端统计策略

* 客户端日志累积20条上传一次；
* 网络环境切换至wifi时，检测待上传日志；
* 标记日志上传标志位，上传成功24小时后清除本地日志；
* 上传失败后，尝试重试，重试若干次清除本地日志。
