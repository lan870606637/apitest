# 亿驾 v4.0+ 客户端交互接口文档


# 一、 背景说明

  本版本接口文档,针对HiTalk项目.

# 二、接口设计原则

## 1. 只支持`HTTP`协议的`POST`请求

## 2. 接口地址组成
```
    http://api.carbit.com.cn/v4.0/{api_name}.json
```
* `api_name`, 接口名称;

`URL`示例
```
    http://api.carbit.com.cn/v4.0/record.json
```

## 3. 请求内容格式

    请求内容均在`Request Body`中记录，以`json`格式。

## 4. 返回内容格式

    返回内容包含`code`，`message`和`context`，格式依`format`参数而定

    返回示例(`json`)

```
{
    "code": 0,          # 返回状态码请参看状态码说明列表
    "message": "OK",
    "context": NULL     # 扩展内容
}
```

## 5. 签名加密方式
* 签名方式

    在`HTTP`请求`header`头中传递`X-SIGN`参数置，用于服务端校验。

    `X-SIGN`生成方式：
    时间戳精确到：秒
    未登录状态下`TOKEN`为空，也可不传递`X-TOKEN`
    由于Android 7.0 开始,存在无法获取IMEI的情况, 所以IMEI该改为可以为空

    ```
    X-SIGN = strtoupper( md5( TOKEN + IMEI + app_key + URI + timestamp ) ),timestamp
    ```
    `URI`示例
    ```
    /v4.0/record.json
    ```

## 6. 终端类型加密方式
* 终端类型加密

    在`HTTP`请求`header`头中传递`X-BIZ`参数配置，用于服务端校验。

    `X-BIZ`对应参数：android、ios (后续可添加wince等)
    ```  
    X-BIZ = android
    ```

## 7. `Token`

   客户端请求登录成功后，服务器段分配客户端一个`X-TOKEN`。
   在`HTTP`请求`header`头中传递`X-TOKEN`参数配置，用于验证客户端操作的合法性。
   客户端退出后`X-TOKEN`为空。

* 内容加密方式

    `Request Body`中的内容均需要经过加密

# 二、接口列表及示例说明

### 1. 用户请求短信密码接口 ###

**公共参数说明**

| 参数         | 是否必须 |     备注    |
| ------      | ------- | ---------- |
| 接口版本     | √       |             |
| 用户手机号   | √        |             |
| X-TOKEN     | ×       |             |

**POST /v4.0/send-sms-password**

**Resource URL**

``` http://api.carbit.com.cn/v4.0/send-sms-password.json ```

**Example Request**

```
{
    "context": {
        "phone_num": "13877073541"
    }
}
```
**Example Response**

**发送成功返回**

```
{
    "code": 0,
    "message": "OK"
}

```

**发送失败返回**

```
{
    "code": 3002,
    "message": "超过每天发送短信条数最大值"
}

```
```
{
    "code": 3015,
    "message": "发送短信失败"
}

```

### 2. 用户登录接口

**公共参数说明**

| 参数         | 是否必须 |     备注    |
| ------      | ------- | ---------- |
| 接口版本     | √       |             |
| 用户手机号     | √       |             |
| 短信密码     | √       |             |
| X-TOKEN     | ×       |             |

* 客户端返回标识符，及用户ID，`TOKEN`值,用户相关信息

**POST /v4.0/login**

**Resource URL**
```
http://api.carbit.com.cn/v4.0/login.json
```

**Example Request 1**

```
{
    "context": {
        "login_type": "PHONE",           #登录方式
        "phone_num": "13877073541",      #用户输入的手机号码
        "sms_password": "110119"         #短信接收到的密码
    }
}
```

**Example Request 2**

```
{
    "context": {
        "login_type": "QQ",                                 		#登录方式 QQ 或者 WECHAT 或者 QIHU
        "third_id": "AE8F9CF55CC967F5B48C61298136334E",       #第三方帐号ID
        "union_id": "SIJEHNDSFSDFDSFFSDF",                           #第三方账号对于 third_id 的补充，特殊平台需要，如 wechat
        "third_nick_name": "卡比特人",                         #第三方昵称
        "third_gender": "男",                               #用户性别，第三方返回“男”，“女“， QIHU返回没有性别字段
        "third_avatar": "http://aliyun-oss/xxxxxxxx"       #已上传到OSS服务器的第三方头像地址.
    }
}
```
**Example Response**
```
{
    "code": 0,
    "message": "OK",
    "context": {
        "user_id": "3130387",
        "token": "1067dc92691747de9939cde7aa439b99",
        "user_info": {
            "name": "卡比特人",
            "gender": "男",
            "user_avatar": "http://down.carbit.lo/avatar/NWxKZDVpcnBad0xM/m/gwm?1521526215",
            "phone_num": "15527461924"
        }
    }
}
```

**登录失败返回**
```
{
    "code": 3001,
    "message": "登录失败"
}
```
**密码失效返回**
```
{
    "code": 3003,
    "message": "短信密码已失效"
}
```
**其他错误**
```
{
    "code": 1010,
    "message": "字段%s不能为空"    包括: 登录类型, 手机号, 密码 不能为空
}
```
```
{
    "code": 3018,
    "message": "非法的手机号码格式"
}
```
